بناء مشاريع ومحفظة

شوف… إذا فيه شيء فعليًا يرفع قيمتك كمبرمج أو طالب تقنية، فهو المشاريع. مو الشهادة، ولا الكلام الحلو اللي تكتبه في الـCV… المشاريع نفسها. شيء يسوّيه مخّك ويدّك، شيء يشتغل قدّام الناس، شيء يثبت إنك فاهم وش قاعد تسوي.

والمشكلة؟ كثير طلاب عندهم مشاريع ممتازة بس طريقة عرضهم لها تضيّع قيمتها. واحد رافع مشروع بدون README… الثاني مشروعه مرتب لكن ملفاته كأنها دولاب ملابس منقلب… والثالث ساحب على الديمو ومو حاط ولا صورة.

واللي يميزك مو كثرة المشاريع… لا. اللي يميزك إن مشروعك يكون واضح، مفهوم، وله هدف، وشكله “نظيف”. خلّني أوريك كيف تبني محفظة ترفعك فوق بين الطلاب.

كيف تختار فكرة مشروع تمشي وتضرب؟

أول خطوة… لا تختار مشروع بس عشان “لازم أسوي مشروع”. اختار شيء تحس إنه فعليًا يحل مشكلة. لو مشروعك يفيد طالب، دكتور، قروب، جهة… صدّقني بيصير أثقل من مشروع ضخم ما له معنى.

أمثلة أفكار قوية:

  • موقع يساعد الطلاب يبحثون عن شيء (جداول، مواد، اختبارات).
  • أداة تسهّل مهمة مملة ومتكررة.
  • مشروع يسحب بيانات من API ويعرضها بشكل مرتب.
  • نظام صغير (Login – Dashboard – إدارة بيانات).

المهم… مشروعك يخدم أحد. لو خدم شخص واحد… يعتبر مشروع ناجح.

رتّب ملفاتك وخلّ الـGitHub يلمع

  • اسم واضح للمشروع:
    ✔ SEU-Planner
    ✘ project88-final-realfinal-v2
  • مجلدات مرتبة:
    frontend / backend / assets / images / docs
  • خلك محترم بالكوميتات:
    ✔ add GPA calculation feature
    ✘ aaa
  • أضف صور داخل المشروع… لأنها أسرع شيء يفهم الشخص مشروعك.

كيف تكتب README يشيل مشروعك فوق؟

الـREADME هو وجه مشروعك… تخيل تروح مقابلة لابس كويس ومتعوب على شكلك؟ README هو نفس الشيء لمشروعك.

وهذه هي الصيغة اللي تخلي مدير التوظيف يكمل للآخر:

  • عنوان واضح: اسم مشروعك وفكرة بسيطة.
  • وصف سريع: المشروع يسوّي إيش؟ وليه سويته؟
  • صور / GIF: أهم نقطة — تنقذ مشروعك.
  • Features: خصائص واضحة.
  • Tech Stack: التقنيات المستخدمة.
  • Installation: طريقة التشغيل.
  • Demo Link: رابط مباشر.
  • What I learned: نقطة قوية خاصة للمقابلات.

أمثلة README — سيئ VS ممتاز

❌ README سيئ:

Project 1  
this is my project.  
use python.

لا وصف… لا صور… لا هدف… لا شيء. هذا النوع يخرب قيمة أي مشروع مهما كان قوي.

✅ README ممتاز:

# SEU Planner
أداة تساعد طلاب الجامعة يصنعون جدول مذاكرة تلقائي للفاينلات بناءً على أيامهم المتاحة.

## 📌 الهدف
كنا كطلاب نضيع وقت طويل كل ترم نحاول نقسم الأيام ونوزّع المواد، فبنيت أداة
تسوي الجدول تلقائي خلال ثواني.

## 🚀 المميزات
- توليد جدول تلقائي
- توزيع المواد حسب الصعوبة
- واجهة سهلة وواضحة
- حفظ البيانات تلقائيًا

## 🛠 التقنيات
HTML - CSS - JavaScript

## 🎯 طريقة التشغيل
1. افتح index.html
2. اختر المواد والأيام
3. يولد لك الجدول مباشرة

## 🎥 صور
(لقطة شاشة هنا)

## 🌐 الديمو
https://fxrg.github.io/SEU-Planner/

## 📚 ماذا تعلمت؟
- تحسين منطق توزيع البيانات
- تنظيم المشروع وترتيب الملفات
- بناء واجهة بسيطة وسهلة الاستخدام

شوف كيف من أول نظرة صار عندك فكرة كاملة عن المشروع؟ هذا هو الهدف.

سوي لك موقع بسيط يجمع مشاريعك

حتى لو صفحة وحدة… موقعك الشخصي يعطي انطباع قوي جدًا. حط فيه:

  • نبذة قصيرة.
  • روابط GitHub و LinkedIn.
  • مشاريعك مع صور ووصف.
  • وسيلة تواصل.

النهاية؟ المحفظة مو عدد مشاريع… بل “جودة العرض + وضوح الفكرة + تنفيذ مرتب”. لو سويت مشروعين ممتازة بتوثيق قوي… أفضل من 10 مشاريع ملخبطة.
